## Analysis of the Source Code

We are provided with a Python script (`chall.py`) that performs two different cryptographic operations:

1. **The PRNG (Mersenne Twister)**
The server asks us to guess a 32-bit number generated by `random.getrandbits(32)`.
• If we guess wrong, the server reveals the correct number.
• If we hit a streak of 3 correct guesses, we unlock the "Jackpot."
• Python's `random` module uses the **MT19937 (Mersenne Twister)** algorithm.
2. **The RSA Samples**
Once the jackpot is unlocked, every subsequent correct guess provides an RSA sample:
• A unique modulus $n$ (product of two 512-bit primes).
• A fixed public exponent $e = 17$.
• A ciphertext $c = m^e \pmod n$, where $m$ is the flag.

# Exploitation Part

Exploit was implemented using `pwntools` for networking, `randcrack` for the PRNG state recovery, and `sympy` for the CRT and $n$-th root calculations.

Execution:

1. **Syncing:** Sent 624 dummy guesses to the server and parsed the "Wrong. The number was X" responses to feed into the `RandCrack` object.
2. **Jackpot:** Used the cracked state to predict the next 3 numbers, satisfying the streak requirement.
3. **Collection:** Continued predicting numbers to trigger the RSA sample output until 17 $(n, c)$ pairs were collected.
4. **Decryption:** Applied CRT and the 17th root to the collected data.
